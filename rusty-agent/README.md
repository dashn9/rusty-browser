# rusty-agent

A single-browser gRPC node deployed as a serverless function via [Flux](https://github.com/fluxfunction/flux). Each instance owns exactly one Chromium browser and exposes a TLS gRPC server for the master to connect to and send commands.

---

## How it works

1. Flux spawns the agent binary with the master's gRPC URL as the first argument
2. Agent launches Chromium with an optional fingerprint identity and geo-matched proxy
3. Agent detects its public and private IP, binds a gRPC server on a free OS-assigned port
4. Agent registers itself with the master (`Register` RPC) — reporting `browser_id`, `public_ip`, `private_ip`, and `grpc_port`
5. Master stores the connection info and begins sending `BrowserCommand` messages directly to the agent over TLS

---

## gRPC Services

See [`rusty-proto/proto/rusty.proto`](../rusty-proto/proto/rusty.proto) for the full message definitions.

**`BrowserAgent`** — served by the agent, called by the master:
- `Execute(BrowserCommand) → CommandResult`

**`Master`** — served by the master, called by the agent on startup:
- `Register(RegisterAgentRequest) → RegisterResponse`

## Supported Commands

| Action | Description |
|---|---|
| `Navigate` | Navigate to a URL |
| `Click` | Click at coordinates (human or direct) |
| `NodeClick` | Click a CSS selector |
| `TypeText` | Type into the active element or a selector |
| `MouseMove` | Move the mouse to coordinates |
| `HumanMouseMove` | Human-like mouse movement |
| `ScrollBy` | Scroll by Y pixels |
| `ScrollTo` | Scroll a selector into view |
| `Screenshot` | Capture the current viewport as base64 |
| `FetchHtml` | Get inner HTML of a selector |
| `FetchText` | Get inner text of a selector |
| `EvalJs` | Evaluate JavaScript and return the result |
| `FindNode` | Check if a CSS selector exists |
| `WaitForNode` | Wait for a CSS selector to appear |
| `CreateContext` | Open a new browsing context (tab) |
| `CloseContext` | Close a browsing context |
| `CloseBrowser` | Shut down the agent process |

## TLS

The agent loads `agent.crt` and `agent.key` from its working directory to serve its gRPC endpoint. These are generated by the master at `/initialize/` and bundled into the agent zip automatically.

For the outbound registration call to the master, the agent loads `master.crt` (also bundled) as the trusted CA. When `--native-tls` is passed (set automatically when `grpc_server_url` is configured on the server), the agent uses the system's native root CAs instead — required when the master is behind a tunnel like ngrok.

## Environment Variables

| Variable | Description |
|---|---|
| `FLUX_EXECUTION_ID` | Set by Flux — used as the agent's execution identity |
| `RUSTY_BROWSER_CONFIG` | JSON-encoded `ChromeBrowserConfig` for driver/binary paths and flags |

---

## Arguments

| Argument | Description |
|---|---|
| `<master_url>` | (required, positional) gRPC URL of the master server |
| `--native-tls` | Use system root CAs to verify the master cert instead of the bundled `master.crt` |
